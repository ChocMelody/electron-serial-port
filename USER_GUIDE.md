# Serial Port HTTP Forwarder 用户指南

## 简介

Serial Port HTTP Forwarder 是一款基于 Electron、Vue 3 和 NestJS 开发的桌面应用程序，专门用于将串口数据实时转发到指定的 HTTP 接口。该应用提供直观的图形界面，方便用户配置串口参数和 HTTP 接口地址，并实时监控数据传输状态。

## 主要功能

- 实时读取串口数据
- 将串口数据转发到指定的 HTTP 接口
- 图形化用户界面，方便配置和监控
- 支持多种串口参数配置（波特率、数据位、停止位、校验位）
- 日志记录和状态监控
- 自动保存用户配置

## 系统要求

- Windows 7 及以上版本
- 至少 2GB RAM
- 至少 100MB 可用磁盘空间

## 安装说明

1. 从发布页面下载最新版本的安装程序 `Serial Port HTTP Forwarder-Setup.exe`
2. 双击安装程序文件
3. 按照安装向导提示完成安装过程
4. 安装完成后，可以在桌面或开始菜单找到应用程序图标

## 界面介绍

应用程序主界面分为两个主要部分：

### 1. 侧边栏菜单
- **配置中心**：用于配置串口参数和 HTTP 接口地址
- **运行日志**：用于查看串口连接状态和数据传输日志

### 2. 主内容区域
根据选择的菜单项显示相应的内容。

## 使用方法

### 配置串口和 HTTP 接口

1. 点击左侧菜单的"配置中心"
2. 在"串口配置"区域：
   - 选择要连接的串口设备
   - 设置波特率（常见为 9600 或 115200）
   - 设置数据位（通常为 8）
   - 设置停止位（通常为 1）
   - 设置校验位（通常为"无"）
3. 在"HTTP 接口配置"区域：
   - 输入目标 HTTP 接口的完整 URL（例如：http://example.com/api/data）
4. 点击"保存配置"按钮保存设置

### 连接和数据转发

1. 点击左侧菜单的"运行日志"
2. 点击"连接"按钮建立串口连接
3. 成功连接后，串口状态将显示为"已连接"
4. 应用程序会自动将接收到的串口数据转发到配置的 HTTP 接口
5. 可在日志窗口查看连接状态和数据传输详情

### 断开连接

1. 在"运行日志"页面点击"断开"按钮
2. 串口连接将被断开，状态更新为"未连接"

## 配置说明

### 串口参数

- **端口**：选择要连接的串口设备（如 COM1, COM2 等）
- **波特率**：数据传输速率，常见值有 9600、19200、38400、57600、115200
- **数据位**：每个数据包的数据位数，通常为 8
- **停止位**：每个数据包的停止位数，通常为 1
- **校验位**：数据校验方式，可选无校验、偶校验、奇校验等

### HTTP 接口配置

- **接口地址**：完整的 HTTP POST 请求地址，应用程序会将串口数据以 POST 请求发送到此地址

## 日志说明

日志窗口显示以下类型的信息：

- **INFO**：一般信息，如配置加载、连接状态变化等
- **SUCCESS**：成功操作，如连接成功等
- **ERROR**：错误信息，如连接失败、数据发送失败等
- **DATA**：接收到的串口数据
- **HTTP**：HTTP 请求相关信息

## 常见问题

### 1. 无法找到串口设备
- 确保串口设备已正确连接到计算机
- 检查设备管理器中是否识别到串口设备
- 尝试点击"刷新"按钮更新端口列表
- 在某些 Windows 系统上可能需要管理员权限才能访问串口

### 2. 连接失败
- 检查串口参数设置是否正确
- 确保没有其他程序占用该串口
- 验证串口线缆是否正常工作

### 3. HTTP 请求失败
- 确保目标 HTTP 接口地址正确且可访问
- 检查网络连接是否正常
- 查看目标服务器是否正常运行

### 4. 数据未转发
- 确认串口连接状态为"已连接"
- 检查串口设备是否正在发送数据
- 查看日志中是否有错误信息
